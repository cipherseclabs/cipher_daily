Weekly Brief – July 29, 2025

Cybersecurity Events & Incidents

  • Active exploitation of Microsoft SharePoint on‑prem servers via zero‑day "ToolShell" (CVE‑2025‑53770/CVE‑2025‑49704) used to deploy web shell (spinstall0.aspx), steal machine keys, and enable remote execution. Over 400 systems compromised, including U.S. federal and state agencies across sectors. Threat actors including Chinese “Linen Typhoon,” “Violet Typhoon,” and Storm‑2603 attributed. Source: https://www.microsoft.com/en-us/security/blog/2025/07/22/disrupting-active-exploitation-of-on-premises-sharepoint-vulnerabilities/
  • Microsoft’s initial July patch failed to fully correct the flaws, resulting in expanded compromise across ~9,000 SharePoint servers globally, including the U.S. National Nuclear Security Administration. Source: https://www.reuters.com/sustainability/boards-policy-regulation/microsoft-knew-sharepoint-security-flaw-failed-effectively-patch-it-timeline-2025-07-22/
  • Possible leak within Microsoft’s MAPP program: suspected that early disclosure to security vendors may have facilitated exploitation by Chinese-linked groups. Microsoft is investigating MAPP integrity. Source: https://www.reuters.com/technology/microsoft-probing-if-chinese-hackers-learned-sharepoint-flaws-through-alert-2025-07-25/
  • Emerging ransomware threat: new “Warlock” ransomware gang (likely descended from Black Basta) is actively targeting vulnerable SharePoint servers using stolen machine keys to maintain persistence after patch. Source: https://www.axios.com/2025/07/29/microsoft-sharepoint-hacks-ransomware

Threat Actors

  • Linen Typhoon (China‑state affiliated): targeting government, defense, education; now exploiting SharePoint flaws. Source
  • Violet Typhoon (China‑nexus espionage group): active in NGO, media, finance, health sectors; involved in SharePoint campaign. Source
  • Storm‑2603: China‑based actor observed chaining SharePoint exploits since early July. Source
  • Warlock ransomware: emerging criminal group (descendant of Black Basta) targeting unpatched SharePoint servers and exploiting stolen machine keys. Source

Analyst Comments / Defender Impact Summary

  • The SharePoint "ToolShell" exploitation exemplifies how patch bypasses and chained zero‑days rapidly escalate risk, particularly with stolen MachineKey artifacts enabling persistence even post‑patch.
  • Organizations must ensure full remediation of both CVE‑49704 and CVE‑53770 and validate patch completeness; static patch status alone may not guarantee protection.
  • Insider leaks via early-access programs like MAPP undermine trust and threat-defender parity; access minimization and vendor audit trails are recommended.

Summary

  • Immediate threats: SharePoint zero‑day exploitation campaign; Warlock ransomware targeting unpatched servers; compromised machine keys enabling persistent access.
  • Actionable steps: Validate application of full SharePoint patches (including latest update for CVE‑53770), rotate ASP.NET MachineKey values, search for spinstall*.aspx artifacts, block and monitor known IOC IPs.
  • Longer‑term needs: Review vendor early‑warning programs like MAPP for leakage risk; enforce stricter NDA and audit controls; invest in detection of post‑exploit behaviors (e.g. machine‑key exfiltration, web shell persistence).
